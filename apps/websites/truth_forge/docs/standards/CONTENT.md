# Content Standards

**Purpose**: Writing guidelines, voice, and tone for the Truth Forge website

**Status**: Active  
**Last Updated**: 2026-01-27

---

## Core Principles

### Truth Forge Voice

**Warm, authoritative, human.**

- **Warm**: Inviting, not cold or sterile
- **Authoritative**: Confident, not arrogant
- **Human**: Real, not corporate speak

### Framework Alignment

All content aligns with the Framework:
- Uses Framework concepts (ME:NOT-ME, HOLD:AGENT:HOLD, TRUTH:MEANING:CARE)
- Speaks to Stage 5 minds
- Reflects Truth Forge as THE SOURCE

---

## Writing Guidelines

### Tone

| Context | Tone | Example |
|---------|------|---------|
| **Hero sections** | Confident, clear | "The source of all systems." |
| **Product descriptions** | Technical, precise | "Stage 5 cognitive architecture deployed as AI." |
| **About content** | Personal, authentic | "We exist for normal people who need accessible technology." |
| **Technical content** | Clear, direct | "HOLD:AGENT:HOLD is the universal pattern." |

### Voice Characteristics

**DO**:
- Use "we" and "you" (human connection)
- Be direct and clear
- Use Framework terminology correctly
- Show, don't just tell

**DON'T**:
- Use corporate jargon
- Over-explain simple concepts
- Apologize for complexity
- Use marketing fluff

## Framework Terminology

### Core Concepts

| Term | Usage | Example |
|------|-------|---------|
| **ME:NOT-ME** | Identity division | "ME provides intent, NOT-ME provides structure." |
| **HOLD:AGENT:HOLD** | Universal pattern | "Every system follows HOLD:AGENT:HOLD." |
| **TRUTH:MEANING:CARE** | The Furnace | "TRUTH becomes MEANING becomes CARE." |
| **Stage 5** | Cognitive model | "Built for Stage 5 minds." |
| **Not-Me** | Product name | "Your Not-Me is your digital extension." |

### Grammar Rules

- **ME:NOT-ME** (colon, ALL CAPS) - Framework principle
- **Not-Me** (hyphen, Normal Caps) - Product name
- **not_me** (underscore, lowercase) - Code/folder reference

## Content Structure

### Page Sections

1. **Hero**: Clear value proposition
2. **Introduction**: What and why
3. **Details**: How it works
4. **Benefits**: What you get
5. **CTA**: Next step

### Headlines

- **H1**: Page title (one per page)
- **H2**: Major sections
- **H3**: Subsections
- **H4+**: Supporting content

**Headline Guidelines**:
- Clear and descriptive
- Include keywords for SEO
- Use Framework terminology appropriately
- Avoid clickbait

## SEO Content

### Meta Descriptions

**Length**: 150-160 characters  
**Include**: Primary keyword, value proposition, call to action

**Example**:
```
Truth Forge: The source of all systems. Stage 5 cognitive architecture 
deployed as AI. Build your Not-Me. Preorder now.
```

### Page Titles

**Format**: `[Page Name] | Truth Forge`

**Examples**:
- `About | Truth Forge`
- `Framework | Truth Forge`
- `What is a Not-Me? | Truth Forge`

## Accessibility

### Alt Text

- **Descriptive**: Describe what the image shows
- **Contextual**: Include relevant context
- **Concise**: Keep under 125 characters when possible

**Examples**:
```
✅ "Truth Forge logo: lightning bolt breaking through circle"
✅ "Jeremy Serna, founder of Truth Forge, speaking at conference"
❌ "Image"
❌ "Photo"
```

### Link Text

- **Descriptive**: Link text describes destination
- **No "Click Here"**: Use meaningful text
- **Context**: Makes sense out of context

## Brand Voice Examples

### Product Description

**Good**:

**Bad**:
> "Our AI assistant helps you be more productive and efficient in your daily tasks."
