## Why Sterile? (The Verification Independence Principle)

Credential Atlas must be **architecturally sterile** to function as a trusted verification layer:

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    THE VERIFICATION INDEPENDENCE PRINCIPLE                   │
│                                                                              │
│   If Credential Atlas could spawn organisms, it would have a direct          │
│   financial incentive in verification outcomes. This destroys trust:         │
│                                                                              │
│   ✗ Could spawn competitors to entities assessed as "not ready"              │
│   ✗ Could suppress certification of entities viewed as threats               │
│   ✗ Could favor entities in which it had economic stake                      │
│   ✗ Trust would be destroyed — the value proposition collapses               │
│                                                                              │
│   REAL-WORLD PARALLEL:                                                       │
│   Credit rating agencies that issue debt would never certify                 │
│   themselves as creditworthy. Credential Atlas follows the same logic.       │
│                                                                              │
│   STERILITY IS NOT A LIMITATION. IT'S THE ARCHITECTURE THAT                  │
│   MAKES THE SYSTEM TRUSTWORTHY.                                              │
│                                                                              │
└─────────────────────────────────────────────────────────────────────────────┘
```

### Five-Layer Sterility Defense

| Layer | Mechanism | What It Prevents |
|-------|-----------|------------------|
| **Genetic** | No `Primitive/seed/` directory | Cannot run reproduction code |
| **Schema** | `reproduction_rights_schema.yaml` validation | Invalid configs rejected |
| **Config** | `SEED_MANIFEST.yaml` defaults to sterile | Must explicitly enable fertility |
| **Approval** | Genesis approval for fertile spawns | Unauthorized fertile spawns blocked |
