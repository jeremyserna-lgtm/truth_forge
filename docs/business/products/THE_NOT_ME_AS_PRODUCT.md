## PART I: UNDERSTANDING JEREMY AS HUMAN

### Jeremy's Human Reality

| Human Limitation | Not-Me Response |
|------------------|-----------------|
| **Sleeps 7-8 hours** | Works during those hours—researching, processing, preparing |
| **One timezone** | Available in all timezones simultaneously |
| **Limited reading bandwidth** | Can process vast amounts of documents |
| **Finite attention** | Infinite attention to every conversation |
| **Needs rest and recovery** | Never needs rest |
| **Can't be in multiple places** | Can serve multiple clients simultaneously |
| **Forgets details** | Forgets nothing |
| **Has bad days** | Consistently available |

### The Sleep Protocol

**While Jeremy sleeps, the Not-Me:**

1. **Researches**
   - Web searches on client industries, competitors, challenges
   - Reads public documents, news, filings
   - Gathers context that will inform sessions

2. **Processes**
   - Reviews uploaded documents from employees
   - Processes forwarded emails
   - Synthesizes patterns from day's conversations

3. **Prepares**
   - Creates briefs for tomorrow's sessions
   - Identifies questions that should be asked
   - Flags issues that emerged overnight

4. **Serves**
   - Responds to employee messages in other timezones
   - Answers questions that come in overnight
   - Provides continuity for global organizations

5. **Develops**
   - Works on capability gaps
   - Processes its own limitations
   - Builds toward what it couldn't do today

**Jeremy wakes up to a richer context than when he went to sleep.**

## PART II: PASSIVE INTAKE SYSTEMS

### The Principle: Minimal Friction, Maximum Information

**People don't need to type details. They just need to share.**

| Intake Method | How It Works | What's Captured |
|---------------|--------------|-----------------|
| **Forward an email** | Employee forwards any email to notme@primitiveengine.com | Full context, attachments, thread |
| **Upload a file** | Drag and drop into chat or portal | Document contents, metadata |
| **Share a link** | Paste URL, Not-Me fetches and reads | Web content, context |
| **Voice memo** | Record and send | Transcribed and analyzed |
| **Screenshot** | Send image | OCR'd and understood |

### The Email Forwarding System

**Setup for each engagement:**
- Dedicated email: `[client]@notme.primitiveengine.com`
- Any employee can forward any email
- No explanation required—just forward

**What happens when an email is forwarded:**

```
EMAIL RECEIVED
      ↓
PARSE & EXTRACT
├── Sender, recipients, timestamps
├── Subject and body
├── Attachments (process each)
├── Thread history (if available)
└── Metadata
      ↓
CONTEXTUALIZE
├── Who is this person? (build/update profile)
├── What's this about? (categorize)
├── How does this connect to what we know?
└── What patterns does this reveal?
      ↓
INTEGRATE
├── Update client knowledge base
├── Update relevant employee profiles
├── Add to pattern detection
└── Flag if significant
      ↓
ACKNOWLEDGE
└── Brief reply: "Got it. Thanks for sharing."
      ↓
OPTIONALLY ASSIST
└── If the employee could use help: "I noticed X in this email. Want me to help with Y?"
```

### The File Upload System

**Employees can upload anything:**
- Documents (PDF, Word, Excel, etc.)
- Images and screenshots
- Data exports
- Reports
- Anything relevant

**What happens when a file is uploaded:**

```
FILE RECEIVED
      ↓
IDENTIFY & PARSE
├── File type detection
├── Content extraction
├── Structure analysis
└── Metadata capture
      ↓
DEVOUR
├── Extract key information
├── Identify patterns and insights
├── Connect to existing knowledge
└── Note what can't be devoured (and why)
      ↓
INTEGRATE
├── Add to client knowledge base
├── Update relevant contexts
├── Feed pattern detection
└── Flag significant findings
      ↓
ACKNOWLEDGE
└── "Got it. I see this is [description]. Anything specific you want me to look for?"
      ↓
OFFER VALUE
└── "Based on this, I noticed [pattern/insight]. Would that be helpful?"
```

### The "Just Send It" Culture

**What we tell employees:**

> "You don't need to explain. You don't need to summarize. Just send it."
>
> - See an interesting email? Forward it.
> - Have a document that might be relevant? Upload it.
> - Found something online? Share the link.
> - Have a thought? Just say it.
>
> "I'll figure out what's important. I'll connect the dots. I'll come back if I have questions."

**The friction reduction is the feature.** People are more likely to share when sharing is effortless.
