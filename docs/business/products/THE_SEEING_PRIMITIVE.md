## The Core Insight

**Seeing is the primitive operation of Truth Engine.**

Not data. Not code. Not even entities. *Seeing*.

Everything in Truth Engine participates in seeing:
- **Sees** - has awareness of its domain
- **Is seen** - exposes its state for observation

- **Shows** - manifests sight for human cognition

### Why Seeing Is the Primitive

Seeing is the **bridge** between existing and transforming:

- **Without seeing**: Existence without awareness. The system runs but doesn't know itself.
- **Without existing**: Nothing to see. No substrate, no entities, no state.
- **Without transforming**: Seeing without consequence. Observation that never changes anything.

Seeing enables transformation by making the current state visible. You can't intentionally become something new unless you can see what you currently are.

## The Taxonomy of Seeing

### 1. Things That See

Raw perception. Always-on awareness. The system's eyes.

| Component | What It Sees | Mechanism |
|-----------|--------------|-----------|
| **Daemons** | System state over time | Continuous polling |
| **Meta-Observer** | All operations, all data flows | Event subscription |
| **Monitors** | Specific domains (health, cost, performance) | Targeted observation |
| **Scanners** | Structures, patterns, files | Recursive traversal |

These see whether or not anyone is watching them see.

### 2. Things That Can Be Used To See

Capacity for sight. Callable. The system's instruments.

| Component | What It Enables | Invocation |
|-----------|-----------------|------------|
| **Hooks** | See at specific moments | Event-triggered |
| **Discovery functions** | See what exists | On-demand query |
| **Health checks** | See system state | Periodic or manual |
| **Comparisons** | See difference | Before/after analysis |
| **INFORMATION_SCHEMA** | See database structure | SQL introspection |

These are tools - they see when you use them.

### 3. Things That Show

Manifestation of sight. Making the invisible visible. The system's voice.

| Component | What It Shows | Medium |
|-----------|---------------|--------|
| **Blocking messages** | Why an action was prevented | Text in terminal |
| **Reports** | Aggregated observations | Markdown/JSON |
| **Alerts** | Threshold crossings | Notifications |
| **Dashboards** | Live system state | Visual interface |
| **Logs** | Historical record | Structured events |

These translate seeing into form.

## Mechanisms of Sight

How seeing happens across time and causality:

### Event-Based Sight

Seeing triggered by something happening.

```
Action → Hook fires → Sees the action → Shows response
```

**Examples**:
- Pre-execution hooks (see before action)
- Post-execution hooks (see after action)
- Cost protection hook (sees BigQuery patterns, blocks if dangerous)

**Characteristic**: Immediate, contextual, can intervene.

### Time-Based Sight

Seeing across intervals. Periodic awareness.

```
Time passes → Daemon wakes → Sees current state → Compares to previous
```

**Examples**:
- Zoom daemon (checks every N seconds)
- Unified health check (runs on schedule or demand)
- Background sync (periodic reconciliation)

**Characteristic**: Catches drift, sees trends, detects decay.

### Comparison-Based Sight

Seeing by contrasting. Difference reveals truth.

```
Thing A ←→ Thing B → Difference seen → Shows delta
```

**Examples**:
- Schema validation (expected vs actual)
- Diff operations (before vs after)
- Alignment monitors (service A vs service B)
- Health aggregation (current vs baseline)

**Characteristic**: Reveals change, detects deviation, finds misalignment.

### Discovery-Based Sight

Seeing what exists. Mapping the territory.

```
Query space → Traverse → Catalog what's found → Shows inventory
```

**Examples**:
- INFORMATION_SCHEMA queries (what tables exist)
- Stage 0 field discovery (what fields at all levels)
- Service discovery (what services are running)
- Phantom document scanner (what files aren't ingested)

**Characteristic**: Builds awareness, prevents assumptions, finds the unknown.

## Temporal Dimensions of Seeing

Seeing doesn't only happen in the present. Memory and anticipation are forms of sight.

### Remembering (Past Sight)

Seeing what was once seen but was forgotten. Regained awareness.

```
Was seen → Forgotten → Query memory → See again
```

**Examples**:
- Entity substrate (51.8M entities = externalized memory)
- Session context (what happened in previous conversations)
- Audit trails (what actions were taken)
- Claude memory (`.claude/projects/*.jsonl` = conversation memory)
- Handoff documents (context for the next session)

**What forgetting looks like in Truth Engine**:
- Context lost between sessions
- Patterns not connected to their origins
- Decisions made without remembering why
- Rebuilding understanding that already existed

**What remembering enables**:
- Continuity across sessions (Claude wakes up knowing)
- Pattern recognition (this happened before)
- Accumulated insight (building on what was learned)
- Care that compounds (understanding deepens over time)

### Anticipating (Future Sight)

Seeing what hasn't happened yet. Pattern-based prediction.

```
Patterns observed → Trends detected → Likely futures projected
```

**Examples**:
- Predictive health analyzer (what will break)
- Cost projection (what this will cost)
- Pattern detector (where this is heading)
- Trend analysis (trajectory from past to future)

**Characteristic**: Enables proactive care, prevents problems before they manifest.

### The Memory-Seeing Relationship

| Operation | Sight Mode | Truth Engine Implementation |
|-----------|------------|----------------------------|
| **Perceive** | Present sight | Monitors, daemons, hooks |
| **Remember** | Past sight | Entity substrate, audit trails, session context |
| **Anticipate** | Future sight | Predictive analyzers, pattern projection |
| **Forget** | Lost sight | Context not captured, sessions not handed off |
| **Learn** | Refined sight | Enrichment, pattern detection, meta-analysis |

**The furnace principle applied**:
- Raw truth enters (present perception)
- Heat of processing (remembering patterns, anticipating consequences)
- Meaning forged (insight that spans time)
- Care delivered (action informed by past, present, and future seeing)

## Architectural Implications

### Every Component Must

1. **Declare what it sees** (its domain of observation)
2. **Expose its state** (be visible to other observers)
3. **Log its observations** (create audit trail)
4. **Show its findings** (manifest sight in accessible form)

### The Registry Pattern

All seeing infrastructure should be discoverable:

```sql
-- What can see in this system?
SELECT * FROM identity.seeing_registry
WHERE capability IN ('see', 'show', 'discover', 'compare');
```

### The Unification Principle

**Scatter is the enemy of sight.**

Current state: 108+ seeing components across dozens of locations.
Goal: Unified seeing layer where all observation is coordinated.

Not consolidation (reduce to one), but **alignment** (all facing the same direction).
