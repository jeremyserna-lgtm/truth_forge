## Strategic Overview

### Brand Architecture Model

The Truth Engine family operates as an **Endorsed Hybrid** architecture—combining the credibility of a unified parent with the flexibility of distinct product brands.

**Why Endorsed Hybrid?**
- Each brand serves a **different audience** with different needs
- The parent brand (Truth Engine) provides **credibility and trust**
- Individual brands can **evolve independently** without compromising siblings
- Customer journeys can **cross brands** naturally (discovery → product → enterprise)
- Acquired or future brands can be **integrated seamlessly**

**Comparable Models:**
- Marriott: Courtyard by Marriott, Ritz-Carlton, W Hotels
- Salesforce: Salesforce, Tableau, Slack, MuleSoft
- Alphabet: Google, Waymo, Verily, DeepMind

## SEO Interconnection Strategy

### Hub and Spoke Model

```
                    truthengine.com (Hub)
                           │
           ┌───────────────┼───────────────┐
           │               │               │
    primitive-engine   credential-atlas   stage5mind
         .com              .com             .com
           │               │               │
           │               │               │
      Technical        Enterprise       Consumer
       Content          Content         Content
```

### Cross-Domain Linking Strategy

**Rule 1: Hub Links Out**
- truthengine.com links to all subsidiaries
- Subsidiaries link back to hub (footer, about)
- Creates clear hierarchy for Google

**Rule 2: Siblings Link Strategically**
- Credential Atlas mentions Primitive Engine for "how we build"
- Stage 5 Mind mentions Credential Atlas for "how we verify"
- Creates logical user journeys

**Rule 3: Concept Sites Feed Ecosystem**
- exist-now.com → educational content → links to all brands
- the-not-me.com → discovery content → links to Stage 5 Mind

## Technical SEO Standards

### Universal Requirements

1. **Schema Markup**
   - Organization schema on all sites
   - Parent organization linking
   - Product schema where applicable

2. **Core Web Vitals**
   - LCP < 2.5s
   - FID < 100ms
   - CLS < 0.1

3. **Mobile-First**
   - All sites responsive
   - Mobile navigation consistent

4. **HTTPS**
   - SSL on all domains
   - HSTS enabled

### Cross-Domain Tracking

**GA4 Configuration:**
- Single GA4 property for ecosystem
- Data streams per domain
- Cross-domain measurement enabled
- User journey tracking across brands

**UTM Convention:**
```
utm_source=[origin_brand]
utm_medium=[content_type]
utm_campaign=[campaign_name]

Example:
stage5mind.com/?utm_source=credential-atlas&utm_medium=footer&utm_campaign=family-nav
```
